项目简介
glsx-bi-report 是广联数星（GLSX）面向汽车零售/门店经营场景打造的 移动端 BI 数据报表 H5 应用。主要在企业微信中打开使用，为总部、区域、督导、销售、驻店员工、运营等不同角色提供数据看板、经营分析与预警推送能力。

技术栈
类别	技术
框架
Vue 3 + TypeScript + Vite
UI
Vant 4（移动端组件库）
状态
Pinia + Vuex
图表
ECharts 5
网络
Axios
监控
Sentry
适配
lib-flexible + postcss-pxtorem
集成
企业微信 / 微信 JSSDK、远程配置中心
核心业务模块
经营分析：总部总览、区域分析、门店列表/详情、二手车专项、流失门店、我的关注等
人效分析：打卡完成、交车目标、日报/周报/月报、录单异常、APP 接待、战败归类、人员行为与业绩异常监测
业绩分析：品类总览、门店维度品类分析
预警推送：驻店/运营/区域预警、实时推送、群推送
销售视角：个人总览、门店与员工分析
人才分析：人才结构与留存等分析
我的事项：业务待办与事项详情
营销战报（README 中另有说明）：众筹、膨胀金、嘀加、车损权益等活动的喜报/战报页面
项目特点
企微优先：生产环境默认要求在企业微信中访问，支持企微 SSO 登录与手机号快捷登录
角色驱动：按督导、销售、驻店、运营等角色动态调整路由与权限
可视化丰富：封装大量 ECharts 组件（折线、柱状、中国地图等）及筛选、搜索等业务组件
远程配置：启动时从配置中心拉取 h5-report 模块配置，支持 VConsole 等运行时开关
目录结构（概览）
src/
├── api/          # 接口层（经营分析、人效、预警、SSO 等）
├── components/   # 通用业务组件（图表、筛选、搜索等 40+ 个）
├── views/        # 页面视图（按业务域划分）
├── router/       # 路由与权限
├── store/        # Pinia 状态（登录、导航等）
└── utils/        # 工具（请求、配置、埋点等）
一句话概括：这是 GLSX 汽车零售 BI 体系的 企微移动端数据门户，集经营看板、人效监测、业绩分析、预警推送和营销战报于一体，供各级管理人员在手机上随时查看业务数据。
