SCRM 微前端项目简介
SCRM（企微管家 SCRM+IOV） 是面向汽车行业的私域流量数字化营销平台，由广联数科（glsx）体系开发，定位为「专注汽车行业数字化 + 智能营销解决方案」，帮助商户将流量转化为销量。项目采用 Qiankun 微前端架构，将大型后台拆分为「主应用 + 多个子应用」，支持独立开发、独立部署、按需加载。

整体架构
┌─────────────────────────────────────────────────┐
│              scrm-base（主应用 / 基座）           │
│  登录 · 权限 · 菜单 · 布局 · 路由 · Qiankun 调度   │
└──────────────────────┬──────────────────────────┘
                       │ 动态注册子应用
       ┌───────────────┼───────────────┐
       ▼               ▼               ▼
 scrm-micro-home  scrm-micro-battles  scrm-micro-points
   首页 / 工作台      营销中心 / 活动       积分核销
主应用 scrm-base：负责登录鉴权、侧边栏菜单、面包屑、全局状态与动态路由；用户登录后，根据后端返回的菜单树动态注册路由，并在 #subapp-viewport 容器中通过 Qiankun 加载对应子应用。
子应用：各自独立仓库、独立构建，以 UMD 格式打包，生产环境部署在 /child/{应用名}/ 路径下，通过 bootstrap / mount / unmount 生命周期接入主应用。
子应用职责
子应用	主要业务
scrm-micro-home
首页工作台、快速上手指南、数据看板（客户/产值/权益等图表）、AI 外呼卡片、精准营销、运营服务等
scrm-micro-battles
营销作战、活动记录、秒杀/众筹活动、订单管理、素材库、公告栏、产品组、支付商户、SaaS 订单转化等
scrm-micro-points
积分核销列表、订单明细、批量结算导入/导出
仓库中目前包含以上 3 个子应用；主应用菜单由后端动态下发，理论上还可扩展更多微应用（菜单 URL 格式为 entry@name）。

技术栈
框架：Vue 2.6 + Vue Router + Vuex + Element UI
微前端：Qiankun 2.x（沙箱隔离、单实例激活、资源预加载）
构建：Vue CLI 4 + Webpack，子应用 externals 共用 CDN 上的 Vue / Element 等
公共能力：glsk-common / glsx-common（HTTP、工具库）、glsx-components（权限按钮、表格等）、Sentry 错误监控、配置中心远程拉取
多版本登录入口：支持标准版（/login）、运营版（/oss）、服务商版（/sp）、冀联版（/jl）及 cgacar 等定制皮肤
核心运行机制
用户登录 → 后端返回权限菜单 → 主应用解析菜单并 动态 addRoute，同时缓存到 Session。
进入 Layout 布局页 → 调用 registerApps(menuList) 向 Qiankun 注册子应用。
主应用向子应用 下发 props：routerBase、用户 Session、路由表、HTTP 实例、消息总线（messenger）、历史记录工具等。
子应用根据主应用下发的路由表动态生成内部路由，实现 菜单驱动、权限驱动 的页面加载。

一句话总结：这是一个基于 Qiankun 的 汽车行业 SCRM 营销后台，主应用统一管理登录、菜单与路由，子应用按业务域（首页、营销、积分）拆分，通过后端菜单动态装配，支持多商户、多产品版本的白标化部署。
